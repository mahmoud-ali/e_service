# Bank Side API Documentation

This API allows the bank to query invoice details and notify the system when a payment is successfully processed.

## Base URL
`/api/v1/invoice/tra/webhooks/bank-callback/`

## Authentication
All requests require a valid API Key sent in the request header.

**Header Name:** `X-API-KEY`
**Value:** `[Your-API-Key]`

---

## 1. Query Invoice Details

Retrieves the details of an invoice using the receipt number.

**Endpoint:** `GET /api/v1/invoice/tra/webhooks/bank-callback/`

### Query Parameters
| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `receipt_number` | string | Yes | The 10-digit receipt number generated by the system. |

### Response (200 OK)
```json
{
  "name": "Miner Name",
  "amount": "25000.00",
  "receipt_number": "0000000001"
}
```

### Error Responses
- **400 Bad Request:** Missing `receipt_number`.
- **404 Not Found:** Receipt number does not exist.

---

## 2. Notify Payment Success

Notifies the system that a payment has been successfully completed.

**Endpoint:** `POST /api/v1/invoice/tra/webhooks/bank-callback/`

### Request Body
```json
{
  "receipt_number": "0000000001",
  "payment_status": "SUCCESS"
}
```

### Body Parameters
| Parameter | Type | Required | Description |
| :--- | :--- | :--- | :--- |
| `receipt_number` | string | Yes | The 10-digit receipt number. |
| `payment_status` | string | Yes | Must be exactly `SUCCESS` to mark the invoice as paid. |

### Response (200 OK)
```json
{
  "status": "Payment recorded successfully"
}
```

### Error Responses
- **400 Bad Request:** 
    - Payment status is not `SUCCESS`.
    - `receipt_number` is missing or invalid.
    - Invoice is not in `Pending Payment` status (e.g., already paid or cancelled).
- **404 Not Found:** Receipt number does not exist.

---

| Code | Description |
